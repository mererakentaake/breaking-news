# 📺 Breaking News TV

A live HLS streaming TV app — **Breaking News by LeadStory NZ** — built with:

| Layer | Tech |
|---|---|
| UI | React 18 + Vite |
| Mobile | Capacitor 6 (Android & iOS) |
| Desktop | Electron 31 |
| CI/CD | GitHub Actions → Android APK |

---

## Prerequisites

| Tool | Version |
|---|---|
| Node.js | 20+ |
| npm | 10+ |
| Java (JDK) | 17 (for Android) |
| Android Studio | Latest (for Android) |
| Xcode | 15+ (for iOS, macOS only) |

## Quick Start

```bash
# 1. Clone & install
git clone https://github.com/YOUR_ORG/breaking-news-tv.git
cd breaking-news-tv
npm install

# 2. Run in browser
npm run dev
```

## Android

### First-time setup
```bash
# Build web assets & add Android platform (run once)
npm run build
npx cap add android
npx cap sync android

# Open in Android Studio
npm run cap:android
```

### Build APK locally
```bash
npm run android:build
# Then in Android Studio: Build → Generate Signed APK
# Or via Gradle:
cd android && ./gradlew assembleDebug
```

## GitHub Actions — Android APK

Every push to `main` / `master` triggers `.github/workflows/android.yml`.

### Secrets required for signed release APK

Go to **GitHub → Settings → Secrets and variables → Actions** and add:

| Secret | Description |
|---|---|
| `KEYSTORE_BASE64` | Base64-encoded `.keystore` file |
| `KEYSTORE_PASSWORD` | Password for the keystore |
| `KEY_ALIAS` | Alias of the signing key |
| `KEY_PASSWORD` | Password for the key |

#### Generate a keystore (first time):
```bash
keytool -genkey -v \
  -keystore release.keystore \
  -alias breaking-news \
  -keyalg RSA -keysize 2048 \
  -validity 10000
```

#### Encode it to Base64:
```bash
# macOS / Linux
base64 -i release.keystore | pbcopy   # copies to clipboard

# Or output to file
base64 -i release.keystore > keystore.b64
```

Paste the output as the `KEYSTORE_BASE64` secret.

> **Without secrets:** The workflow falls back to a **debug APK** automatically. This is safe for PRs and forks.

### Release APK via tag
```bash
git tag v1.0.0
git push origin v1.0.0
# → GitHub Action builds signed APK and creates a Release
```

## Notes

- HLS playback uses [hls.js](https://github.com/video-dev/hls.js/) loaded from CDN; Safari uses native HLS.
- Android `allowMixedContent: true` is set in `capacitor.config.ts` to allow the HLS stream over HTTPS.
- The Android workflow **does not commit** the `android/` folder — run `npx cap add android` locally first, then commit the generated folder.
